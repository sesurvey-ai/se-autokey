# se-autokey

> 📘 สำหรับการพัฒนาต่อ/แก้บั๊ก: อ่าน [DEVELOPMENT.md](DEVELOPMENT.md) —
> รวมความรู้ระบบทั้งสองฝั่ง บั๊กที่แก้แล้ว และ backlog ทั้งหมด

ดึงข้อมูลเคลมประกันรถยนต์จาก **ISURVEY** (cloud.isurvey.mobi) แล้วกรอกลงระบบ
**EMCS / eClaim** (eclaim3.blueventuregroup.co.th) อัตโนมัติในคำสั่งเดียว
พร้อมโหลดรูปภาพ จัดชื่อรูปด้วย template matching และอัปโหลดขึ้น EMCS

> แปลงมาจาก workflow เดิมใน `selenium.ipynb` (ยังเก็บไว้เป็นข้อมูลอ้างอิง)

## ติดตั้งครั้งแรก (สำหรับนักพัฒนา)

```powershell
pip install -r requirements.txt
copy .env.example .env     # แล้วเปิด .env ใส่ username/password ทั้งสองระบบ
```

ต้องมี Google Chrome ติดตั้งอยู่ (Selenium จัดการ chromedriver ให้เอง)

> 👥 **ติดตั้งให้พนักงาน (ไม่ต้องลง Python):** ดูหัวข้อ "ติดตั้งให้พนักงาน" ด้านล่าง — ใช้ Python ที่ฝังมากับโฟลเดอร์

## ติดตั้งให้พนักงาน (deploy — copy แล้วรันได้เลย ไม่ต้องลง Python)

โฟลเดอร์นี้พก Python มาในตัว (`runtime\` = Python 3.13 embeddable + dependencies ครบ)
เครื่องพนักงานจึง **ไม่ต้องลง Python และไม่ต้องรัน setup ใดๆ** — แค่มี Chrome แล้ว copy ไปวาง

**ครั้งแรก ทำบนเครื่องที่มีเน็ต ครั้งเดียว (สร้างตัว runtime):**
1. ดับเบิลคลิก **`build-runtime.bat`** → โหลด Python embeddable + ลง dependencies ลงโฟลเดอร์
   `runtime\` (~250 MB) — ถ้ามี `runtime\` อยู่แล้ว/เคยรันแล้ว ข้ามข้อนี้ได้

**deploy ไปเครื่องพนักงาน:**
2. ลง **Google Chrome** บนเครื่องพนักงาน (ถ้ายังไม่มี)
3. copy **ทั้งโฟลเดอร์** ไปวาง — **ต้องมี `runtime\` ติดไปด้วย**
   (ตัด `runs/`, `downloaded_images/`, `__pycache__/` ออกได้ สร้างใหม่เอง — แต่ **ห้ามตัด `runtime\`**)
4. ใช้งาน: ดับเบิลคลิก **`start-webui.bat`** / คู่มือพนักงานอยู่ที่ **`README_พนักงาน.txt`**

> - ครั้งแรกที่รันบนแต่ละเครื่อง Selenium จะโหลด chromedriver ให้ตรงเวอร์ชัน Chrome อัตโนมัติ (ใช้เน็ตครั้งเดียว แล้วแคชไว้)
> - `.env` มีรหัสบัญชี **กลาง** (ISURVEY/EMCS/ISURVEY_REPORT) — copy ไป = ทุกเครื่องใช้บัญชีเดียวกัน
>   (ใช้ได้ เพราะ "คนคีย์" ที่แจ้ง ISURVEY มาจากเลขท้ายเลขเคลม ไม่ใช่บัญชีที่ login) — อย่าแชร์โฟลเดอร์ออกนอกทีม
> - ต้องวางที่เครื่องที่ใช้จริง (Chrome เปิดที่เครื่องนั้นเพื่อตรวจ+กดส่ง — รันรวมเครื่องเดียวให้คนอื่นเปิดผ่าน browser ไม่ได้)
> - `start-webui.bat` จะใช้ `runtime\` อัตโนมัติ ถ้าไม่มีจะ fallback ไปใช้ Python ของเครื่องแทน
>   (กรณีนั้นต้องลง Python 3.10+ เอง แล้วรัน **`setup.bat`** ลง dependencies ก่อน)

## วิธีใช้แบบหน้าเว็บ (ง่ายสุด — แนะนำ)

ดับเบิลคลิก **`start-webui.bat`** (หรือสั่ง `python webui.py`) จะเปิดหน้าเว็บ
ที่ `http://127.0.0.1:8765` ให้เอง — ใส่เลขเคลมในช่อง กดปุ่ม **"รันโปรแกรม"**
แล้วดู log การทำงานสดๆ บนหน้าจอ

- ใส่ได้ทีละเคลมหรือหลายเคลม (บรรทัดละเลข) / ติ๊ก "อ่านอย่างเดียว" หรือ
  "ไม่ยุ่งกับรูปภาพ" / เลือกความเสียหาย เบา–หนัก ได้
- **ข้อมูลจาก ISURVEY ไม่ครบ?** ถ้ากรอกอัตโนมัติไม่ได้ (field บังคับว่าง/
  dropdown ไม่โหลด) จะ **ไม่ error ทิ้ง** แต่ขึ้นกล่องเหลือง "ต้องกรอกข้อมูลเอง"
  พร้อมบอกว่าช่องไหน — ไปกรอกในหน้า EMCS (Chrome) แล้วกด "ดำเนินการต่อ"
  มันจะทำงานต่อให้ (ดูหัวข้อ "หยุดรอเมื่อข้อมูลไม่ครบ" ด้านล่าง)
- เป็นแค่ "ตัวเปิดโปรแกรม" — เบื้องหลังเรียก `main.py` ตัวเดิมผ่าน subprocess
  ทุกอย่างจึงเหมือนรันใน terminal เป๊ะ (Chrome เปิดให้เห็น, บันทึกเป็น draft,
  **ไม่กดปุ่ม "ส่งงานใหม่" ให้**)
- รันได้ครั้งละ 1 งานเท่านั้น (ISURVEY จำกัด 1 บัญชี/การใช้งาน)
- ใช้ไลบรารีมาตรฐานของ Python ล้วน ไม่ต้องติดตั้งอะไรเพิ่ม

```powershell
python webui.py                 # เปิดที่พอร์ต 8765 + เปิดเบราว์เซอร์ให้
python webui.py --port 9000     # เปลี่ยนพอร์ต
python webui.py --no-open       # ไม่เปิดเบราว์เซอร์ให้อัตโนมัติ
```

## วิธีใช้แบบ command line

```powershell
# flow เต็ม: อ่าน ISURVEY → โหลดรูป → จัดชื่อรูป → กรอก EMCS → อัปโหลดรูป
python main.py --claim 2026013105763

# ถ้าค้นหาแล้วมีหลายแถว ให้ระบุเลขเซอร์เวย์ด้วย
python main.py --claim 2026013105763 --invoice SEABI-213260100295

# อ่านข้อมูล + โหลดรูปอย่างเดียว (ยังไม่กรอก EMCS)
python main.py --claim 2026013105763 --read-only

# ทำหลายเคลมรวดเดียว (batch เต็มรูปแบบ: อ่าน + กรอก EMCS ทีละเคลมจนจบชุด
# login ครั้งเดียว, เคลมไหนพลาด/ไม่ใช่เคลมแห้ง/มีเรื่องแล้ว = ข้ามพร้อมเหตุผล,
# จบด้วยตารางสรุป ✅/⏭️/❌ + เลข e-Survey — แล้วเข้า EMCS ไล่กดส่งงานเอง)
python main.py --claims-file claims.txt        # บรรทัดละเคลม: "เลขเคลม [เลขเซอร์เวย์]"
python main.py --claims 2026013105763,2026013105999

# batch แบบอ่านอย่างเดียว (ไม่แตะ EMCS)
python main.py --claims-file claims.txt --read-only

# กรอก EMCS จากข้อมูลที่เคยอ่านไว้ (ไม่ต้องเปิด ISURVEY ใหม่)
python main.py --data-json runs/2026013105763.json

# ตัวเลือกอื่น
python main.py --claim ... --skip-images        # ไม่ยุ่งกับรูปเลย
python main.py --claim ... --loss-type ""       # ไม่เลือกลักษณะความเสียหาย (ไม่ใช่เคลมแห้ง)
python main.py --claim ... -y                   # ไม่หยุดถามก่อนกรอก EMCS
```

## การอ่านข้อมูล: ผ่าน API เป็นค่าเริ่มต้นแล้ว

ปกติระบบ**อ่าน ISURVEY ผ่าน HTTP API** (เร็วระดับวินาที ไม่เปิด browser ฝั่งอ่าน
ไม่มี DOM race) — โหลดรูปผ่าน `get-images` API ด้วย
(รายละเอียด endpoint: [tools/isurvey_api_findings.md](tools/isurvey_api_findings.md))

```powershell
# อ่าน + โหลดรูป ไม่เปิด browser เลย (ค่าเริ่มต้น = API)
python main.py --read-only --claim 2026013046414

# เทียบ scrape ⟷ API ทีละ field (ตรวจว่ายังตรงกัน)
python main.py --compare --claim 2026013046414

# บังคับใช้วิธีเดิม (Selenium scrape เปิด browser) — fallback ถ้า API มีปัญหา
#   หรือเมื่ออ่านเคลมสดที่ต้องการคู่กรณี/ผู้บาดเจ็บ/ทรัพย์สินครบ
python main.py --scrape --read-only --claim 2026013046414
```

> ตรวจแล้ว: ข้อมูลตรง scrape 8 เคลมแห้ง + รูปตรง zip 27/27 + กรอก EMCS ครบ 2 เคลม (draft)
> ฝั่งเขียน EMCS ยังเป็น browser เหมือนเดิม
> **คู่กรณี/ผู้บาดเจ็บ/ทรัพย์สิน (เคลมสด) API ยังไม่อ่าน → ใช้ `--scrape`** (เคลมแห้งไม่ต้องใช้;
> เคลมสดถูกบล็อกจากการกรอก EMCS ตามนโยบายอยู่แล้ว)

## ส่งงาน + แจ้งสถานะกลับ ISURVEY (ปุ่มเดียวบนหน้าเว็บ)

หลังบอทกรอก EMCS เป็น draft เสร็จ ระบบจะ **หยุดรอให้ตรวจ** แล้วโชว์ปุ่ม
**"✅ ส่งงาน + แจ้ง ISURVEY"** บนการ์ดงานนั้น:
1. คุณตรวจ draft ในหน้าต่าง EMCS (Chrome) ให้เรียบร้อย (ความเสียหาย >8 รายการ เติมให้ครบก่อน)
2. กดปุ่ม **"✅ ส่งงาน + แจ้ง ISURVEY"** → บอทกด "ส่งงานใหม่" ใน EMCS ให้ +
   แจ้งกลับ ISURVEY (`EMCSstatus=send`, คนคีย์ดูจากเลขท้ายเลขเคลม)

> **Safety:** บอท **ไม่กดส่งงานเอง** จนกว่าคุณจะกดปุ่ม — ถ้าไม่กด = เก็บเป็น draft
> ทำเฉพาะงานสถานะ **"รายงานสร้างใหม่"** (ปุ่ม "ส่งงานใหม่" มีเฉพาะ draft) —
> งานที่ส่งไปแล้ว (ประกันตรวจสอบ*) จะไม่ถูกแตะ
> ต้องตั้ง `ISURVEY_REPORT_USER`/`ISURVEY_REPORT_PASS` ใน `.env` ก่อนใช้

> CLI สำรอง (แจ้ง ISURVEY เรื่องที่กดส่งใน EMCS เองแล้ว): `python main.py --report-isurvey [--dry-run] --claim X`

## บันทึกงานลงฐานข้อมูลกลาง se-key + กันทำซ้ำ

เชื่อมกับระบบทะเบียนงาน **se-key** (`key.sesurvey.cloud` — Chrome extension + Express/SQLite
ที่ทีมใช้บันทึกว่าเลขเซอร์เวย์ไหนคีย์โดยใคร ส่ง iSurvey แล้วหรือยัง) เพื่อให้ "งานที่บอททำ"
โผล่ในทะเบียน/รายงานเดียวกับงานที่คีย์มือ:

1. **กันทำซ้ำ (ก่อนกรอก EMCS):** ตรวจเลขเซอร์เวย์กับ se-key DB ก่อน — ถ้ามีอยู่แล้ว
   (ไม่ว่าสถานะไหน) จะ **ข้าม ไม่กรอก EMCS** + แจ้งเหตุผล (batch = ⏭️ ในตารางสรุป /
   เคลมเดียว = หยุดก่อนถามยืนยัน)
2. **บันทึกงาน (ตอนกดส่ง):** เมื่อกด **"✅ ส่งงาน + แจ้ง ISURVEY"** (จังหวะเดียวกับที่แจ้ง iSurvey)
   จะ POST งานเข้า se-key DB ด้วย แล้ว mark `isurvey_sent=1` (se-autokey แจ้ง iSurvey เองแล้ว) —
   `keyer` = คนคีย์ตามเลขท้ายเคลม (`keyer_for`), **ประเภทงานเลือกได้บนหน้าเว็บ** (ดูหัวข้อถัดไป), upsert กันแถวซ้ำ

### เลือกประเภทงาน (งานต้น/งานตาม/SESV/งานรวม) ตอนส่ง

กล่อง "✅ ส่งงาน + แจ้ง ISURVEY" มีแผงเลือกประเภทงาน (ยกตรรกะจาก se-key extension มาไว้ในหน้าเว็บ
ที่เราคุมเอง) — default อัตโนมัติ + ผู้ใช้ปรับเองได้ (โมเดล "มีคนมอนิเตอร์"):

- radio **งานต้น / งานตาม / SESV** — default: SESV ถ้าเลขเซอร์เวย์ขึ้นต้น `SESV`, ไม่งั้น `งานต้น`
  (งานต้น vs งานตาม แยกด้วย `ddlAdd_No` ที่ webui มองไม่เห็น → ผู้ใช้เลือกเอง)
- checkbox **งานรวม** + กรอกเลข invoice (SEABI) ได้หลายเลข → สร้างหลาย row
  (1 primary + N followup) เหมือน extension
- **SESV** ล็อกคู่ "งานรวม" อัตโนมัติ + บังคับกรอก invoice; ตอนแจ้ง iSurvey ใช้ SEABI ตัวแรกแทนเลข SESV
- ตรรกะ: [`sekey_client.build_payloads`](autokey/sekey_client.py) (ลอกจาก `content.js`);
  console (ไม่มีหน้าเว็บ) = ส่งด้วย default งานต้น/SESV

ตั้งค่าใน `.env` (ไม่ตั้งทั้งคู่ = ปิดฟีเจอร์นี้ทั้งหมด ไม่กระทบ flow เดิม):
```
SE_KEY_API_URL=https://key.sesurvey.cloud
SE_KEY_API_KEY=<ค่าเดียวกับ SE_KEY_API_KEY ฝั่ง se-key server>
```

> - ตรวจซ้ำพลาด (เน็ตล่ม/auth) = **ทำงานต่อ** (fail-open) ไม่บล็อกงานหลัก — มี ⚠️ ใน log
> - โค้ด: [`autokey/sekey_client.py`](autokey/sekey_client.py) (REST client) + hook ใน `main.py`
>   (`_sekey_dup_skip` ก่อนกรอก + `_offer_submit` ตอนส่ง)

## เลือกรูปก่อนอัปโหลดเข้า EMCS (หน้าเว็บ)

ก่อนถึงขั้นอัปโหลดรูป ระบบจะ **หยุดรอ + โชว์รูปทั้งหมดเป็นแกลเลอรี** บนการ์ดงาน
ให้ติ๊กเลือกเฉพาะรูปที่จะนำเข้า EMCS แล้วกด **"⬆️ อัปโหลดรูปที่เลือก (N)"**:

- **จัดกลุ่มตามหมวด**: 🚗 รูปรถประกัน (INS) / 📄 เอกสาร-ใบรับงาน (REPORTS) / 📎 อื่นๆ (OTHERS)
  — แต่ละหมวดมี checkbox "เลือกทั้งหมวด" (เช่น ติ๊กออกเอกสารทั้งก้อนในคลิกเดียว)
- เริ่มต้น **ติ๊กครบทุกรูป** (มีปุ่ม "เลือกทั้งหมด / ไม่เลือกเลย") — รูปที่ไม่ติ๊กจะไม่ถูกอัปโหลด
- ไม่เลือกเลย = ข้ามการอัปโหลดรูปทั้งหมดของเคลมนั้น
- โหมด command line (ไม่มีหน้าเว็บ) ยังอัปโหลดทุกรูปเหมือนเดิม (ไม่ถาม)
- กลไก: ตอนโหลดรูปเขียน `_categories.json` (ชื่อ→หมวด) + ตอนตั้งชื่อเขียน `_rename_map.json`
  (ชื่อใหม่→ชื่อเดิม) → [`browser.wait_for_image_select`](autokey/browser.py) อ่านมาจัดกลุ่ม ส่ง marker
  → webui เสิร์ฟรูปผ่าน `/image` → ส่งรายชื่อที่เลือกกลับ stdin → [`emcs.upload_images(only=...)`](autokey/emcs.py)

## เคลมสดที่มีคู่กรณี

ระบบกรอกรถคู่กรณีให้อัตโนมัติ (สูงสุด 20 คัน) จากข้อมูล XML ของ ISURVEY:
เจ้าของ/รถ/ผู้ขับขี่/ใบขับขี่/บริษัทประกันคู่กรณี/เลขกรมธรรม์/ค่าเสียหาย/KFK
- จังหวัด+อำเภอเลือกตามรหัส ISURVEY ที่ map ตรงกับลำดับ dropdown ของ EMCS
  (มี log แสดงชื่อที่เลือกให้ตรวจทุกครั้ง)
- ลักษณะความเสียหายเลือกอัตโนมัติ: ไม่มีคู่กรณี→เคลมแห้ง,
  ประกันผิด→ชนคู่กรณีเสียหาย, คู่กรณีผิด→ถูกคู่กรณีชน (override ได้ด้วย --loss-type)
- ส่วนคู่กรณีของ EMCS ถูกปลดล็อกหลังบันทึกหน้าหลักครั้งแรก — สคริปต์จัดลำดับให้แล้ว
- ผู้บาดเจ็บ/ทรัพย์สิน ยังต้องกรอกเอง (สคริปต์จะเตือน)

## โมเดลความปลอดภัย

การ "บันทึก" ทุกหน้าใน EMCS เป็นแค่ draft ที่แก้ไขได้ — สคริปต์จึงกดบันทึก
ให้ครบทุกหน้า: หน้าหลัก → รถคู่กรณี → ความเสียหาย → Update → อัปโหลดรูป →
**บันทึกหน้าค่าใช้จ่าย**

จุด commit จริงคือปุ่ม **"ส่งงานใหม่"** ที่หน้าค่าใช้จ่าย — **สคริปต์ไม่กดให้
เด็ดขาด** (ตัวค้นหาปุ่มกรองคำว่า 'ส่งงาน' ออกตั้งแต่ต้นทาง) ตรวจสอบบน
browser ที่เปิดค้างไว้ แล้วกดส่งงานเองเมื่อพร้อม

จุดที่หยุดรอคน: ก่อนเริ่มกรอก EMCS จะแสดงสรุปข้อมูล + รายงานความครบ
แล้วรอกด Enter (ข้ามด้วย `-y`) / dropdown เลือกด้วย fuzzy matching —
คะแนนต่ำกว่า 60 มี ⚠️ เตือนใน log

## หยุดรอเมื่อข้อมูลไม่ครบ

เมื่อข้อมูลจาก ISURVEY ไม่ครบ และถึง field **บังคับ** ที่กรอกอัตโนมัติไม่ได้
(เช่น ยี่ห้อรถ/ประเภทรถ/จังหวัด-อำเภอ ว่าง หรือ dropdown โหลดไม่ขึ้น) —
สคริปต์จะ **ไม่ error จบโปรแกรม** แต่จะ **หยุดรอให้กรอกเอง** แทน:

- **หน้าเว็บ:** ขึ้นกล่องเหลือง "ต้องกรอกข้อมูลเอง" บอกชื่อช่อง → ไปกรอก/เลือก
  ในหน้าต่าง EMCS (Chrome) ให้เรียบร้อย → กดปุ่ม **"ดำเนินการต่อ"** สคริปต์ทำงานต่อ
- **command line (มี console):** ขึ้นข้อความ ⏸️ ในเทอร์มินัล → กรอกใน Chrome →
  กลับมากด **Enter** ที่หน้าต่าง
- **รันแบบไม่มีคนเฝ้า (stdin ปิด):** ไม่ค้าง — ข้ามช่องนั้นไปก่อนแล้วเตือนให้กรอกเอง
  ภายหลัง (ระบบ validation ตอนบันทึกจะฟ้องช่องที่ยังขาดอยู่ดี)

field ที่ถือว่า "บังคับ" อิงตาม `CRITICAL_FIELDS` ใน `autokey/claim_data.py`
ส่วน field รองที่ว่าง (เช่น สีรถ/ความสัมพันธ์ผู้ขับขี่) ยัง **ข้ามเงียบๆ เหมือนเดิม**
ไม่รบกวน

## โครงสร้างโปรเจกต์

```
main.py                  จุดเริ่ม — รวม flow ทั้งหมด
start-webui.bat          เปิดหน้าเว็บ (ใช้ runtime\ ที่ฝังมา — ไม่ต้องลง Python)
build-runtime.bat        สร้าง runtime\ ครั้งเดียว (โหลด Python embeddable + ลง deps)
runtime/                 Python 3.13 embeddable + dependencies (พกมากับโฟลเดอร์, ~250MB)
autokey/
  config.py              โหลด credentials จาก .env + path ต่างๆ
  browser.py             Chrome driver + helper (รอ element, fuzzy dropdown, วันที่ พ.ศ.)
  claim_data.py          โครงสร้างข้อมูลเคลม (บันทึก/โหลด JSON ได้)
  isurvey.py             ฝั่งอ่าน: login → ค้นเคลม → อ่าน Tab 1,2,3,7,8 → โหลดรูป
  emcs.py                ฝั่งกรอก: login → สร้างงาน → กรอกทุกส่วน → รูป → ค่าใช้จ่าย
  images.py              โหลดรูป / เก็บกวาดโฟลเดอร์ / จัดชื่อรูป
  processing.py          template matching จัดชื่อรูป (ย้ายมาจาก processing.py เดิม)
templates/check.jpg      รูป template ใบรับงาน
downloaded_images/<เลขเคลม>/   รูปของแต่ละเคลม (รูปรอบเก่าถูกย้ายเข้า _old/ อัตโนมัติ)
runs/<เลขเคลม>.json      ข้อมูลที่อ่านได้ในแต่ละรอบ (ใช้กับ --data-json ได้)
runs/logs/               log ทุกรอบรัน + screenshot/HTML อัตโนมัติเมื่อเกิด error
tools/dump_tabs.py       สคริปต์สำรวจโครงสร้างหน้า ISURVEY (ใช้ตอนพัฒนา)
```

## รายงานความครบของข้อมูล

หลังอ่านแต่ละเคลมจะมีรายงานท้ายสรุป:
- `❌ field สำคัญที่ยังว่าง` — field ที่ EMCS ต้องใช้ ควรตรวจก่อนกรอก
- `⚠️` — ความผิดปกติ เช่น ไม่มีรายการความเสียหาย / เกิน 8 รายการ
- `ℹ️ field รองที่ว่าง` — ขาดได้ ไม่กระทบการกรอก
